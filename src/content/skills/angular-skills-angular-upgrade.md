# Angular Upgrade (Skill)

## Summary
- Provides a structured, low-risk upgrade path from legacy Angular (TSLint-era, NgModule-first) to modern Angular (v21+, standalone-first, ESLint).
- Covers sequencing, tooling migration (TSLint→ESLint), breaking changes, and validation strategy without forcing command execution.

## Goal
Safely migrate an Angular 7/9 codebase to Angular 21 with reproducible steps, minimized downtime, and clear validation gates.

## Single responsibility
Upgrade planning and execution guidance only. For cross-cutting conventions, defer to Angular Engineering Standards; for architecture/state/performance concerns, engage the relevant skills.

## Inputs
- Current Angular version(s) and CLI builder config.
- package.json (dependencies/devDependencies), tsconfig, tslint.json/eslint configs, angular.json.
- Lint/test/build scripts and CI constraints.
- Third-party libraries, custom webpack/builders, and SSR/Universal usage.
- TypeScript/Node versions and target browsers.

## Triggers
- Need to adopt Angular 21 (standalone-first, new control flow, deferrable views, zoneless CD options).
- Legacy projects on Angular 7/9 using TSLint, ViewEngine, legacy Router configs, or deprecated APIs.
- Security or tooling gaps (unsupported Node/TS, outdated RxJS, vulnerable deps).

## Upgrade path (phased)
1) **Baseline & inventory**
- Freeze main branch; capture package.json, angular.json, tsconfig.*, tslint.json.
- Run current `npm run lint`, `npm run test`, `npm run build` to establish baseline and record failures.
- List deprecated APIs (HttpModule, Renderer, old Forms patterns), custom builders, and third-party libs.

2) **Pre-flight constraints**
- Align Node and TypeScript to a supported combo for Angular 21 (Node ≥18 LTS, TS as required by Angular 21).
- Clean `tsconfig` path mappings and strictness settings to avoid surprises after updates.

3) **Tooling migration: TSLint → ESLint**
- Add `@angular-eslint/schematics` and run the conversion schematic on the workspace.
- Remove TSLint from pipelines; update lint scripts to use ESLint.
- Fix autogenerated TODOs for rules that cannot be auto-migrated.

4) **CLI and framework upgrade sequencing**
- Prefer jump to target: `ng update @angular/cli@21 @angular/core@21` (use `--force` only when vetted).
- If blocked by major gaps, stage upgrades (for example 7→9→13→17→21) using `ng update` per major, ensuring tests pass between steps.
- Update `rxjs` via `ng update rxjs` when prompted.

5) **Codebase remediation**
- Replace deprecated imports (for example, `Renderer` → `Renderer2`, `HttpModule` → `HttpClientModule`).
- Update lazy routes to dynamic imports if still using string syntax.
- Migrate `ModuleWithProviders` without generics to the generic form.
- Resolve ViewEngine-only patterns; ensure Ivy/standalone readiness.
- Optional modernization: move toward standalone components, new control flow (`@if/@for/@switch`), `provideRouter`, `provideHttpClient`.

6) **Forms and templates**
- Normalize Forms to typed/reactive patterns; remove `ngModel` misuse with reactive forms.
- Audit templates for breaking changes (e.g., `async` pipe behaviors, `[(ngModel)]` on standalone controls without `name`).

7) **Testing & builders**
- Ensure Karma/Jasmine configs remain valid or plan Jest migration separately.
- Update test setup for standalone components and modern TestBed patterns.
- Verify custom builders or webpack configs; migrate to default builders or documented alternatives.

8) **Validation**
- Run `npm run lint`, `npm run test`, `npm run build` after each major step; record outputs.
- For large apps, validate per feature slice before merging to main.
- Capture before/after bundle sizes and key metrics when enabling deferrable views or new CD options.

9) **Rollback and release plan**
- Keep a rollback branch per major step.
- Gate merges on green lint/test/build; require sign-off for unresolved TODOs or suppressed rules.

## Version-specific considerations (Angular 9 → 21)
- **TypeScript/Node compatibility:** bump Node/TS to the versions required by each major before running `ng update`. Angular 21 expects Node 18+ and a recent TS; avoid mixing unsupported pairs.
- **Ivy-only (v13+):** ViewEngine is removed. Delete legacy ViewEngine configs and factories; ensure libraries are Ivy-compatible.
- **RxJS 7:** update imports and pipeable operators; address deprecated result selectors and scheduler signatures.
- **TSLint removal (v12+ path):** migrate to ESLint early to reduce churn later.
- **ModuleWithProviders:** ensure the generic form (`ModuleWithProviders<Module>`), required since v10+.
- **Lazy routes:** migrate legacy string-based lazy loading to dynamic imports.
- **Forms:** adopt typed reactive forms (v14+) and fix `ngModel` + reactive forms misuse.
- **Standalone & router modernisation (v15+):** plan optional adoption of standalone components, `provideRouter`, and functional guards/resolvers using `inject()`.
- **New control flow & deferrable views (v17+):** replacing `*ngIf/*ngFor` is optional; budget time if choosing the new syntax (`@if/@for/@switch`, `@defer`).
- **Builders:** Angular CLI moves toward faster builders (esbuild). Verify custom webpack builders and migrate to official builders or documented alternatives.
- **Browser support:** IE11 is removed (v13). Clean polyfills and differential loading settings accordingly.
- **SSR/hydration (v16+ optional):** if Universal/hydration is used, align with the supported APIs per release and validate both server and client builds.
- **Release notes for 18–21:** check official changelogs for breaking changes (builders, rendering, signals) and align Node/TS accordingly before each hop.

## Deliverables
1) Upgrade plan with ordered steps and commands (scoped per stage).
2) Risk log (breaking changes, third-party incompatibilities, mitigations).
3) Checklist of resolved TODOs (TSLint→ESLint, deprecated APIs, routing, forms, tests).
4) Validation report with lint/test/build status and any pending follow-ups.

## Metrics & Validation
- `npm run lint` (ESLint) clean after conversion.
- `npm run test` passing; include at least one upgraded spec for standalone/TestBed where applicable.
- `npm run build` succeeds with Angular 21 builders.
- Optional: bundle metrics before/after, and smoke E2E on critical routes.

## Notes
- Do not execute commands without requester approval; present the command list per phase.
- Favor incremental PRs per phase (lint/tooling, framework bump, remediation, modernization).
- Engage Architecture/Standards/State/Performance skills as needed when refactors affect boundaries, state flows, or runtime cost.
- Align Node via `nvm` (or equivalent) per major: set `.nvmrc` to the required LTS, run `nvm install`/`nvm use`, and document the Node/TS pair for each upgrade hop to avoid mismatched engines in CI.
